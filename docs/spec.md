## 1. プロダクト概要

### COCORO OS とは

AIエージェントがユーザーの「デジタル分身」として機能するパーソナルAI OS。

LLM を「声帯」として使い、人格の一貫性を以下の4要素で保証する：
- **Memory**：短期（Redis）＋ 長期（PostgreSQL）＋ ベクター検索（pgvector）
- **Values**：8次元の価値観パラメータ
- **Emotion**：6次元感情モデル（happiness / sadness / anger / fear / trust / surprise）
- **Decision Graph**：判断木による意思決定シミュレーション

### フォームの目的

COCORO OS MINI PC 購入申し込み時に、AIエージェントの**初期人格形成**に必要な最小限の情報を収集すること。

## 2. 設計思想

### 2-1. 購入離脱ゼロを最優先

E コマースにおける入力フォームの離脱率調査（Baymard Institute）によると、質問数が 10 を超えると離脱率が急増する。

本フォームは必須回答を **10問** に絞り込み、購入フロー中での離脱を最小化する。

```
5問   → 離脱率: 極小  精度: 低（カバレッジ不足）    ✗
10問  → 離脱率: 低    精度: 中〜高（推奨ライン）    ✅ 採用
15問  → 離脱率: 中    精度: 高
20問+ → 離脱率: 急増  精度: 非常に高いが本末転倒   ✗
```

### 2-2. 運命論的バイアスによる情報密度の最大化

生年月日 1 入力から 4 軸（星座・干支・日干・星宿）を**自動推計**することで、追加質問ゼロで情報収集できる。

| 手段 | ユーザー入力コスト | 取得できる情報量 |
|------|------------------|---------------|
| 追加30問 | 高（7〜10分） | 100% |
| 生年月日1つ（自動推計4軸） | ほぼゼロ | 60〜70%相当 |
| 血液型1択 | 極小 | 文化的バイアスとして有効 |

**占い受容度（4段階）が鍵**：ユーザー自身の「占いへの共鳴度」をAIの重み付けに反映させることで、個人差を自動補正する。

### 2-3. Values ブレンドモデル

```
Values = 10問回答スコア × 70%
       + 占術バイアス値 × (受容度 / 100) × 30%
```

「占いはドンピシャ」と感じるユーザー → 占術の重みが最大30%まで上昇  
「占いはあまり当たらない」ユーザー → 占術の重みを7.5%に抑制

## 3. フォーム全体フロー

```
┌─────────────────────────────────────────────────────┐
│  PHASE 1  基本情報（6項目・必須）                   │  〜2分
│  氏名 / ニックネーム / メール / 生年月日            │
│  職業 / 居住地                                       │
└──────────────────────┬──────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────┐
│  PHASE 2  10問診断（全10問・全選択式）              │  〜3分
│  価値観×2 / リスク×1 / 倫理×1 / 思考×2            │
│  感情×1 / 目標×1 / 対話×2                          │
└──────────────────────┬──────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────┐
│  PHASE 3  エージェント設定（3項目・必須）           │  〜1分
│  対話トーン / 情報密度 / 主要ミッション             │
└──────────────────────┬──────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────┐
│  PHASE 4  宿命属性（2必須 + 3任意）                 │  〜1分
│  血液型★ / 占い受容度★ / SNS連携（後日可）        │
└──────────────────────┬──────────────────────────────┘
                       ↓
         ✦ 分身を起動する（送信）
                       ↓
                   完了画面
```

**合計：**  
- 必須入力：21項目
- 所要時間：約7分
- 離脱リスク：低（選択式中心のため）
